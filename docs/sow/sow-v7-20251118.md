## 2. Annex A — Project Specifications

### 2.1 Background and Technical Objectives

Trace3 has been engaged to build a **Minimum Viable Product (MVP)** to support sales forecasting through a modernized data architecture, integrating **WideOrbit**, **Informatica**, and **AdBook** data. Focus areas:

- Pipeline optimization
- Improved granularity
- Enhanced refresh cadence

Trace3 will deliver a **production-grade Pacing dashboard** to demonstrate value.

### 2.2 In-Scope

**Data Sources:**
- WideOrbit
- AdBook
- Informatica MDM repository
- Dynamics (source data limited to output of a single existing query provided by Client)

**Work Location:**
- 3540 Toringdon Way, Charlotte, NC 28277
- Remote work permitted for Services not requiring onsite presence

### 2.3 Phases & Scope Particulars

#### Phase 0: Foundation & Framing (1 week)
- Project initiation, kickoff pre-requisites, environment docs, prep checklists
- Kickoff meeting: validate scope, governance, schedule, change request process, contacts, prerequisites
- Project plan + schedule
- Recurring feedback cadence

#### Phase 1: Discovery & Requirements (2 weeks)
- Document knowns, unknowns, assumptions
- Evaluate/recommend initial tech choices (cloud, data platforms)
- Confirm existing platform commitments/constraints
- Flag integration points (ingestion, transformation, visualization)
- Establish in/out of scope guardrails
- Deliver use case definition/validation and MVP success criteria

#### Phase 2: Architecture Design (2 weeks)
- Stakeholder interviews: consumption needs, cadence, KPIs
- End user persona representation
- Confirm technology stack
- Assess current WideOrbit + AdBook data flow
- Document pain points (manual effort, refresh lag, data structure)
- Deliver high-level architecture diagram + tech stack doc

#### Phase 3: Build & Integration (4 weeks)
- Configure data pipelines
- Transform data to support pacing logic
- Create tables with joins, filters, historical granularity
- Provide initial data validation results

#### Phase 4: Dashboards Development (2 weeks)
- Design/develop MVP Pacing dashboard
- **Limit:** new dashboard development limited to recreation of up to **three (3) legacy dashboard pages** of medium-to-high complexity
- Define refresh schedule and business rules for metrics

#### Phase 5: Testing, Feedback & Handoff (2 weeks)
- Support UAT
- Finalize documentation
- Project closure meeting
- Completion Certificate + Client sign-off

### 2.5 Out of Scope
- Data migration or protection services not detailed here
- Support not detailed here
- Open maintenance/support issues unless explicitly included
- Any task or deliverable not stated in Scope of Work

### 2.6 Key Assumptions
- T&E budgeted for **two (2) site trips for two (2) resources** (capped at $10,000, preapproved)
- Services delivered continuously from start through completion/allotted time (excluding holidays/PTO)
- Normal business hours: 8 AM – 5 PM Client local time
- Services to be delivered by End Date or within 12 months of Effective Date ("Expiration Date"); renegotiation via Project Change Request if extending
- All systems in scope must be fully licensed prior to engagement start
- Trace3 will complete Project work remotely when feasible (Webex, Teams, or VPN)

### 2.7 Responsibilities

**Trace3:**
- Written reports + execution documentation
- Provide a lead contact (Trace3 Project Lead)
- Consistency in Trace3 Project team; knowledge transfer on staffing changes

**Client:**
- Provide existing documentation + access within **3 business days** of kickoff
- Make knowledgeable personnel available promptly on request
- Provide an environment ready at Project Start Date
- Responsible for business/data application testing and data backup
- Ensure documentation is accurate/complete/up-to-date
- Grant Trace3 limited license to use Client systems/data as needed
- Assign a lead contact (Client Project Lead)
- Delay attributable to Client beyond 5 business days → fee adjustment for Trace3's additional out-of-pocket expenses
